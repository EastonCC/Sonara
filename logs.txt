Accent-Insensitive Search
=========================

Summary
-------
Search was too strict — "tell" would not match a track titled "U could tëll"
because Django's icontains does exact character matching. Added Unicode
normalization so accented characters (ë, é, ñ, etc.) are treated as their
plain equivalents (e, e, n) during search.


Files Changed
-------------

1. backend/accounts/views.py
   - Added _strip_accents(text): uses unicodedata.normalize('NFKD') to
     decompose characters like ë into e + combining diaeresis, then strips
     the combining marks. Works for all Unicode diacritics.
   - Added _fuzzy_match(haystack, needle_words): checks if every search
     word appears in the accent-stripped version of the target text
   - Rewrote SearchView to use a two-pass approach:
     1. Loose DB query using icontains on the first word to narrow candidates
     2. Python-side accent-insensitive filtering on all words
   - Examples: "tell" matches "tëll", "cafe" matches "café", "n" matches "ñ"

Workstation New Project State Reset
====================================

Summary
-------
When navigating to the Create page (/workstation) and clicking "Create New
Track", the DAW was still showing tracks and project names from the previous
session. This happened because the zustand store only reset the project name
and server ID, but not the actual tracks or editing state.

Root Cause
----------
In Workstation.tsx, the useEffect that runs when there is no projectId only
called setProjectName() and setServerProjectId(). The tracks array, undo/redo
history, piano roll state, and clip selection all persisted from whatever was
loaded before.

Fix
---
Now calls loadProjectData() with a clean default state:
  - projectName: '[Untitled]'
  - bpm: 120, key: C, time signature: 4/4
  - tracks: [] (empty — no preloaded demo tracks)
  - Editing state (selected clips, piano roll, undo/redo) all cleared

Files Changed
-------------

1. frontend/src/workstation/Workstation.tsx
   - Replaced the else branch of the project load useEffect (lines 106-113)
   - Now calls loadProjectData() with empty defaults instead of just
     setting projectName and serverProjectId
